# Prompt Engineering and Loop Engineering Study Project

## Goal

Build top-tier practical judgment and output capability in prompt engineering and loop engineering.

The working definition:

- Prompt engineering: control one model call through instructions, context, examples, constraints, and output contracts.
- Loop engineering: design a repeatable AI workflow that can observe, act, validate, learn from traces, and improve through evals.

## Daily Time

Default: 3-4 hours per day during the B sprint.

If actual time is below 90 minutes on any day, the day converts to a compressed mode:

1. Active recall.
2. One concept.
3. One artifact edit.
4. One eval or review note.

## Success Criteria

By the end of Phase B:

- Explain prompt contract, context contract, tool contract, loop contract, trace, eval, grader, and failure taxonomy.
- Convert vague tasks into structured run contracts.
- Diagnose whether a failure came from prompt, context, tool, state, model, validation, or workflow.
- Produce a reusable prompt/loop/eval template pack.

By the end of Phase C:

- Build a small eval-driven AI workflow.
- Maintain traceable test cases and failure-mode labels.
- Show measurable before/after improvement.
- Produce a Codex-ready handoff and task pack for future automation.
