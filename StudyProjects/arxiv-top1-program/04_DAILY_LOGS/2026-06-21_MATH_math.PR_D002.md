## Core Explanation Notes

### What is it?

Conditional Probability 问的是：在某个条件已经成立的世界里，目标事件发生的比例。Bayesian Update 问的是：新证据出现后，我应该如何更新对目标事件概率的估计。

### Core theory

```text
P(A | B) = P(A and B) / P(B)
```

Bayes 的数学核心：

```text
P(A|E) ∝ P(E|A) * P(A)
```

完整归一化形式：

```text
P(A|E) = P(E|A)P(A) / [P(E|A)P(A) + P(E|not A)P(not A)]
```

Bayesian thinking 的实用决策链条：

```text
prior -> likelihood -> posterior -> EV / tail risk / permission gate
```

注意：evidence quality 不属于 Bayes 公式本体，它属于 evidence reliability、calibration、data quality 和 source audit 层。

### How should I use it?

任何 agent、交易系统或研究系统看到新证据后，都不能直接说“概率提高，所以行动”。必须先问：

1. Prior 是多少？
2. Evidence 是什么？
3. Evidence 是否可靠、是否偏样本、是否过期？
4. `P(A|Evidence)` 怎么估计？
5. Posterior 提高后，EV 是否为正？
6. Tail risk 是否可承受？
7. 这个 posterior 只够提醒、建议，还是够执行？

### Decision rule

```text
posterior probability updates belief;
action permission requires EV + cost + tail risk + authorization + rollback + audit.
```
