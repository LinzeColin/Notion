## Core Explanation Notes

### What is it?

概率不是“主观觉得可能”，而是把不确定现实压缩成可计算结构的方法。它不保证下一次结果，只描述一类结果在规则、数据或模型下如何分布。

### Core theory

最小链条是：现实有不确定性 -> 列出可能结果 sample space -> 定义事件 event -> 用 random variable 把结果映射为数字 -> 用 distribution 表示可能性结构 -> 用 expected value、variance、tail risk 支持决策。

### How should I use it?

以后看到任何“胜率、置信度、预测概率、相似度、策略成功率”，先问：

| Check | Question |
|---|---|
| Sample space | 这个概率覆盖哪些可能结果？有没有漏掉关键状态？ |
| Event | 这里的“成功/上涨/正确”具体定义是什么？ |
| Random variable | 现实结果被映射成了哪个可计算数字？ |
| Distribution | 概率来自历史频率、模型假设、主观估计还是校准后的预测？ |
| Expected value | payoff 和 loss 是否让策略仍然赚钱？ |
| Tail risk | 最坏情况、连续亏损和极端事件会不会摧毁系统？ |

### How it helps decisions

交易里，70% 胜率如果每次赚 1、输 10，仍然可能是负期望。agent 里，80% task success 如果 20% 失败会下错单、删错文件或污染数据库，就不是可自动执行系统。工业系统里，报警预测准确率高，但漏报一次重大事故，整体决策价值可能为负。

### ROI-max action

任何系统先不要问“准确率高不高”，先写出：

```text
Event = 什么算成功
Random Variable = 把现实结果映射成什么数
Payoff = 成功赚多少/节省多少
Loss = 失败亏多少/造成什么损害
Expected Value = P(success)*Payoff - P(failure)*Loss - cost
Tail Risk = 最坏情况是否可承受
```
