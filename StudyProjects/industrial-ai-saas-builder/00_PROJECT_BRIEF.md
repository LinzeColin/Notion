## Core Goal

Build a high-ROI cross-domain capability stack for creating, evaluating, and commercializing an industrial AI SaaS product, using `RAG + SaaS + business strategy + architecture/PM + Codex + AI Software Engineering`.

The working product context is:

> Industrial / rotary kiln intelligent decision SaaS.

This project does not aim to make the user a general expert in every subfield. The realistic Top 1% target is a cross-domain advantage: better than most operators at translating an industrial problem into an AI product, business case, system architecture, implementation plan, and validation loop.

## Acceleration Rule

There is no artificial `D000` or `Day 0`. If the user starts before a planned Day 1, the project starts early and the next real lesson becomes `D01`. If a lesson is completed and the user continues, the next lesson can be pulled forward. Acceleration never skips active recall, teaching, output, GitHub sync, Notion sync, verification, external review handshake, state update, personalization update, or next-plan update.

## Success Criteria

| Capability | Target |
|---|---|
| RAG judgment | Can explain and evaluate retrieval quality, grounding, chunking, embedding, and hallucination failure modes. |
| SaaS judgment | Can define ICP, workflow wedge, pricing hypothesis, retention logic, and MVP boundary. |
| Industrial strategy | Can connect rotary kiln / manufacturing workflows to customer pain, data, ROI, and adoption constraints. |
| Architecture / PM | Can produce PRD, C4-style architecture, data flow, acceptance criteria, risk register, and Codex task pack. |
| AI SWE | Can reason about evals, observability, reliability, deployment, safety, and maintainability. |
| Final output | Can decide whether the product deserves MVP investment using a business + technical decision memo. |
