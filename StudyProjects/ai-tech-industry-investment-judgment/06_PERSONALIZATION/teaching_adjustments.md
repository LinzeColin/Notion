# Teaching Adjustments

## Default Style

Use zero-base chain first, then expert framing:

observable problem -> minimum unit -> representation -> mechanism -> system flow -> constraints/costs -> failure modes -> use cases -> ROI action -> verification.

## Project-Specific Rules

| Rule | Reason |
|---|---|
| Do not start with tickers. | Tickers create premature investment conclusions before evidence and valuation. |
| Explain every metric as a decision tool. | Metrics are useful only if they change action or confidence. |
| Force反方观点. | Prevents narrative lock-in. |
| Use one theme per day. | Prevents shallow coverage across too many AI/finance topics. |
| Separate trend and investability. | The user must learn why a correct trend can still be a bad investment. |
