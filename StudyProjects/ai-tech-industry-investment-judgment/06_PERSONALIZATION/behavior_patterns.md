# Behavior Patterns

Created: 2026-06-21

| Signal | Current interpretation | Adjustment |
|---|---|---|
| User supplied a detailed RTF plan | User wants execution, not another vague strategy discussion. | Use the RTF as locked project input unless contradicted. |
| User emphasizes ROI and Top 1% | Plans must compress toward judgment and output, not passive reading. | Every week has one visible deliverable. |
| Investment topic is high stakes | Avoid trade instructions and unsupported claims. | Keep learning/research/simulation boundary explicit. |

## Open Tracking

- actual minutes if the user provides them;

- source quality mistakes;
- whether the daily output was created;
