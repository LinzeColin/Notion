# Top 1% Strategy

## Top 1% Capability Definition

本项目的 Top 1% 不是“预测股票一定涨”，而是建立跨技术、产业、财务、估值、概率和工具的研究判断力。

| Dimension | Top 1% behavior |
|---|---|
| 判断力 | 能把事实、趋势、假设、市场预期和反方证据分开。 |
| 技术理解 | 能解释 AI 能力提升来自算力、数据、算法效率、产品化和评测变化，而不是只看 benchmark 分数。 |
| 产业推演 | 能追踪瓶颈迁移：GPU -> HBM -> 网络 -> 数据中心 -> 电力 -> 并网 -> 融资或需求。 |
| 财务能力 | 能把收入增长、CapEx、折旧、FCF、ROIC、融资稀释放到同一个模型里。 |
| 估值能力 | 能用 Reverse DCF 反推出市场已经计入的增长和利润率。 |
| 概率能力 | 能用 Bull/Base/Bear、概率、期限、证伪条件管理不确定性。 |
| 输出能力 | 能写出别人可以审查、反驳、复用的 Memo，而不是情绪化观点。 |
| 工具能力 | 能用 Codex/Notion/数据表把研究流程做成可复利系统。 |

## 20% High-Leverage Knowledge

| Concept | Why high ROI |
|---|---|
| AI Scaling | 判断能力进步是否来自更多训练算力。 |
| Algorithmic efficiency | 判断同样能力所需算力是否下降。 |
| Benchmark validity | 防止把刷榜当真实能力。 |
| Agent task horizon | 判断 AI 能否完成更长任务，不只看短问答。 |
| GPU/HBM/network bottlenecks | 找到 AI 基础设施中供给最慢的环节。 |
| Data center power and grid | 判断资本开支是否被物理约束限制。 |
| Supply elasticity | 判断为什么某些瓶颈资产能获得利润池。 |
| CapEx cycle | 判断增长是否会转化为产能过剩或折旧压力。 |
| FCF and ROIC | 判断公司是否真正创造资本回报。 |
| Reverse DCF | 判断市场价格已经包含什么未来。 |
| Bull/Base/Bear | 把观点变成可比较情景。 |
| Falsification condition | 明确什么证据会让自己改错。 |
| Evidence ladder | 管理信息源可靠性。 |
| Memo writing | 把研究转成可审查输出。 |
| Monitoring dashboard | 让研究变成长期系统，不靠临时热情。 |

## 80% Defer List

| Defer | Reason |
|---|---|
| 短期期权 | 高损耗、高杠杆、与学习目标不匹配。 |
| 复制 13F | 持仓滞后、成本和风险承受能力不同。 |
| 全面追逐 AI 新闻 | 信息噪声大，容易替代真正研究。 |
| 同时覆盖几十个行业 | 会变成低深度资料搬运。 |
| 技术分析短线指标 | 不能训练本项目核心的技术-产业-财务判断。 |
| 大型量化交易平台 | 会把学习偏离到工程搭建和交易执行。 |
| 争论唯一 AGI 年份 | 对真实决策价值低，容易制造伪确定性。 |
| 全面学习所有 AI 论文 | ROI 低，应围绕能力测量和产业链问题选择论文。 |

## Ability Tree

| Layer | Content |
|---|---|
| Foundation | 事实/推断/观点分离、证据等级、概率和证伪纪律。 |
| Technical | Scaling、算法效率、benchmark、Agent 长任务能力。 |
| Industrial | GPU、HBM、网络、数据中心、电力、并网、冷却、供应链。 |
| Company | 商业模式、客户、收入、成本、CapEx、现金流、ROIC。 |
| Valuation | Reverse DCF、三情景估值、预期差。 |
| Portfolio | 模拟组合、仓位、相关性、最大损失、风险预算。 |
| Output | Memo、图谱、数据库、Dashboard、口头答辩。 |

## Contrarian Rules

| Common mistake | Correct rule |
|---|---|
| 产业增长等于投资收益 | 投资收益 = 实际结果 - 市场已计入预期。 |
| 公司收入增长等于股东回报 | 必须看 FCF、ROIC、稀释、债务和再投资效率。 |
| AI benchmark 进步等于商业价值 | 必须看任务、成本、可靠性、工作流嵌入和客户付费。 |
| 数据中心需求增长必然利好所有链条 | 利润会迁移，供给释放后瓶颈会变化。 |
| 新闻越多越重要 | 一手资料、可验证数据和反方证据优先。 |

## Investment Safety Note

本项目只产生学习、研究和模拟组合材料。任何真实投资动作都需要独立核验、风险承受能力评估和合规判断。
