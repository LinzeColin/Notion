## Concept: Quant Agent Workspace

### Plain meaning

Quant Agent Workspace 不是“让模型直接下单”的脚本，而是一套把研究、策略、验证、风控、纸面执行、审计和审批串起来的工作系统。

### System components

1. Data Layer: 行情、财务、新闻、公告、宏观、组合持仓。
2. Evidence Layer: 来源、时间戳、可信度、冲突记录。
3. Research Agent: 找机会、写假设、列反方证据。
4. Strategy Agent: 把假设转成可测规则。
5. Validation Lab: 回测、成本冲击、PBO、DSR、walk-forward。
6. Risk Engine: 组合限额、仓位、回撤、相关性、流动性、熔断。
7. Paper Executor: 只跑 simulated / paper trading。
8. Order Intent Queue: 生成订单意图，不直接成交。
9. Human Approval Gateway: 人工确认后才可能进入真实 broker。
10. Audit Log: 每个决策、数据、参数、输出都能 replay。

### Failure modes

- 数据过期但 Agent 继续生成建议。
- 策略只是过拟合参数搜索结果。
- 回测没有成本、滑点和冲击。
- 单策略风险合格但组合层面相关性爆炸。
- Agent 误把 paper trading 成功当成 live-ready。
- 订单路径没有人工审批或 kill switch。

## Workspace Charter v1

### Mission

建立一个能独立运行研究和纸面验证的量化交易 Agent Workspace，用于发现、验证和跟踪策略，但不允许无人监管实盘成交。

### Allowed autonomous actions

| Action | Autonomous? | Notes |
|---|---|---|
| Fetch public market/fundamental data | yes | Must log source and timestamp |
| Generate strategy hypotheses | yes | Must include falsification criteria |
| Run backtests | yes | Must include cost/slippage assumptions |
| Run paper trading | yes | Paper/simulated only |
| Generate risk report | yes | Must include portfolio constraints |
| Generate `OrderIntent` | yes | Intent only, not order submission |
| Submit live order | no | Requires human approval outside this workspace |

### Prohibited actions

- Store live broker secrets in repo.
- Submit real-money order automatically.
- Modify live risk limits automatically.
- Disable kill switch.
- Treat paper trading result as proof of live profitability.

## Kill Conditions v1

| Condition | Trigger | Required action |
|---|---|---|
| Data stale | key data older than allowed freshness window | halt signal generation |
| Validation failed | PBO/DSR/walk-forward gate fails | quarantine strategy |
| Risk breach | position/drawdown/exposure exceeds limits | block order intent |
| API abnormal | broker/data API errors or inconsistent responses | stop paper executor |
| Price anomaly | price jump/spread/liquidity outside limits | block new intents |
| Audit gap | missing source, parameter, or decision log | block promotion |
