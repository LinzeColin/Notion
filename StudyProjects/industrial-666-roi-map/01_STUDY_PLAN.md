# Study Plan

Project: `industrial-666-roi-map`
Duration: 90 days
Progress format: `第X/90天`

## Learning Strategy

The project uses a two-speed system:

| Speed | Purpose | Rule |
|---|---|---|
| Foundation speed | Build the taxonomy, scoring model, source map, and industry-analysis template. | No ranking claim without source strategy. |
| Attack speed | Study high-ROI small classes and produce practical memos. | Every lesson must create a reusable decision asset. |

## Zero-Base Cognitive Chain

Every lesson starts from this chain:

| Step | Industrial version |
|---|---|
| Observable problem | What industrial decision, opportunity, cost, risk, or supply-chain question does this knowledge solve? |
| Minimum unit | The GB/T small class or a specific product/process/customer unit inside it. |
| Representation | Code, name, parent category, score, product list, process flow, value-chain position, risk tags. |
| Mechanism | How materials, equipment, process, labor, energy, customers, standards, and capital turn into value. |
| System flow | Upstream resources -> core process -> quality control -> customer application -> service/maintenance -> profit and risk. |
| Constraints/costs | Capex, regulation, safety, energy, yield, precision, labor, certification, customer switching cost, data transparency. |
| Failure modes | Mistaking size for opportunity, policy for profit, heat for commercial viability, technology for investability. |
| Use cases | Career choice, startup wedge, investment screen, research memo, content thesis, consulting angle, AI product opportunity. |
| ROI action | One table, memo, map, checklist, interview question set, or data-verification task. |
| Verification | User can explain boundary, value creation, bottleneck, opportunity, risk, and personal entry point. |

## 3-Day Launch

| Day | Focus | Teaching object | Output | Verification |
|---:|---|---|---|---|
| D01 | Taxonomy and scoring foundation | Why small class is the minimum unit, and how ranking avoids fake opportunity | Source checklist + scoring schema v0.1 | Can explain why 666 small classes cannot be replaced by broad industry names. |
| D02 | First top-100 ranking batch method | How to score opportunity, trend, ROI, personal fit, and risk deduction | Top-100 ranking workflow + first sample rows | Can explain why hot sectors do not automatically rank high. |
| D03 | Industry memo template | How to turn one small class into a usable decision memo | Small-class deep-analysis template v0.1 | Can distinguish product, process, customer, bottleneck, and personal entry path. |

## Daily Output Contract

Each daily lesson must produce at least one:

| Output type | Example |
|---|---|
| Concept card | "Why small class beats big industry label" |
| Decision table | Opportunity score vs risk deduction |
| Industry memo | One small class deep memo |
| Value-chain map | Upstream, core process, downstream, bottleneck |
| Opportunity screen | Career/startup/investment/research/content actions |
| Verification checklist | What evidence proves the opportunity is real |

## Scoring

| Score | Meaning |
|---:|---|
| 1 | Can recognize the term only. |
| 2 | Can repeat definitions but cannot use the concept. |
| 3 | Can explain boundary and basic mechanism. |
| 4 | Can apply it to a real industry decision or opportunity screen. |
| 5 | Can teach it, compare it with adjacent industries, and detect false claims. |
