## Core Goal

建立面向个人学习者的工业小类级产业判断系统。目标不是机械背诵 666 个工业小类，而是用最高 ROI 的方式获得以下能力：

| Capability | Target |
|---|---|
| 产业边界判断 | 能看懂每个小类生产什么、不生产什么、上下游是谁、和相邻小类有什么区别。 |
| 工业底层语言 | 能理解产品、工艺、设备、材料、质量指标、成本结构、供应链瓶颈。 |
| 机会识别 | 能从职业、创业、投资、研究、内容输出、咨询、AI 产品化角度识别机会。 |
| 真假趋势判断 | 能区分真实产业升级、短期热点、概念炒作、周期反弹、结构性机会。 |
| 跨行业迁移 | 能把一个小类的工艺、设备、材料、客户、成本逻辑迁移到其他小类。 |
| 个人转化 | 能找到适合个人、小团队、AI 放大的切入点，而不是只看重资产叙事。 |

## First-Phase Boundary

This 90-day phase does not promise to deeply master all 666 small classes. That would create shallow coverage and low ROI. The first phase target is:

| Layer | Output |
|---|---|
| Full-map layer | Build the 666-small-class schema, scoring model, source strategy, and ranking workflow. |
| Priority layer | Produce a defensible first ranking for at least the top 100 small classes. |
| Deep-study layer | Deeply study the top 30 small classes or high-leverage clusters. |
| Transfer layer | Build reusable industry-analysis templates, decision rules, and opportunity screens. |
| Memory layer | Maintain GitHub state and lightweight Notion notebook sync when connector access is available. |

## Success Criteria

| Criterion | Acceptance |
|---|---|
| Taxonomy evidence | Official or authoritative source references are recorded before final ranking claims. |
| Ranking rigor | Each ranked item has code, name, parent category, score, opportunity, risk, and personal entry path. |
| Top 30 depth | Each top-30 small class has a usable industry memo, not a generic overview. |
| ROI usefulness | Every lesson ends with how to use the knowledge for decision, work, learning, productivity, or opportunity detection. |
| Personalization | User confusion, preferences, and weak concepts are recorded and used to adjust later lessons. |
| Notion cleanliness | Notion remains a clean notebook. No existing workspace is overwritten. |

## Operating Rules

| Rule | Meaning |
|---|---|
| Concept first | Do not start from industry buzzwords. Build from observable problem, minimum unit, representation, mechanism, cost, failure mode, use case, ROI action, verification. |
| Evidence before ranking | Never fabricate 666-class rows, market size, policy support, enterprise ranking, production data, or technical progress. |
| One main object per lesson | Each daily deep lesson focuses on one small class or one necessary scoring/taxonomy mechanism. |
| Chinese notes | Learning notes, logs, Notion content, and summaries are Chinese by default; professional terms can remain English. |
| GitHub memory | This folder is the source of truth for state, logs, metrics, and handoff. |
| Notion role | Notion is notebook and visualization, not the source of truth. |
