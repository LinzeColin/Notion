# Study Preferences

## Explanation Preferences

| Field | Current rule |
|---|---|
| Language | Chinese by default. Keep English only for professional terms, API names, exact titles, or classification codes. |
| Depth | Detailed, professional, concrete, and operational. Avoid slogans. |
| Teaching style | Start from zero-base cognitive chain, then move to expert judgment. |
| Examples | Prefer business, manufacturing, investment, AI product, productivity, and decision examples. |
| Interaction | Use active recall, compact questions, tables, and default recommendations. |

## Motivation Patterns

| Pattern | Rule |
|---|---|
| User dislikes checklist burden | Agent must self-check sync, state, Notion status, and next plan. |
| User wants ROI | Every lesson must convert knowledge into action or judgment. |
| User wants top-tier capability | Avoid shallow coverage and fake completion. |

## Current Weak Areas To Watch

| Area | Evidence | Intervention |
|---|---|---|
| Fake understanding from labels | User repeatedly asks for deep explanation instead of tasks only. | Explain mechanism and use case before asking output questions. |
| Scope overload | 666 small classes can become shallow if treated as a checklist. | Use ranking, clusters, and top-30 deep dives before full traversal. |
| Notion pollution risk | User wants clean notebook. | Sync concise notes only after reading page format. |
